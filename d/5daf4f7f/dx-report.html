Findings

3つの前提が覆った

ヒアリングを進める中で、当初の仮説と実態が異なる重要な発見がありました。
これらはすべて、今後の戦略設計に直接影響します。

Finding 01

ANDPADの受発注モジュールは「未契約」だった

仮説 契約済みだが現場に定着していない
→
実態 受発注モジュールは完全に未契約
受発注モジュールの追加契約と、協力会社1,500社への段階的な展開が必要です。先に業務フローを整理してから投資判断する方が合理的です。
Finding 02

基幹システムは外部連携ができない構造

仮説 パッケージ製品で、他システムとの連携が可能
→
実態 2015年にフルスクラッチ開発。つなぎ口が存在しない
推測として、クラウド上のカスタムアプリ（ネット上のサービス型ではなく、サーバーを借りて動かしている形態）の可能性があります。基幹システムを前提にしない戦略設計が必要です。
Finding 03

会計は基幹システムとは別の製品（SMILE）

仮説 会計機能は基幹システムの一部
→
実態 大塚商会の別製品「SMILE」を使用
データの流れが「ANDPAD → 基幹 → SMILE」の3段階で、すべて手作業連携です。原価をリアルタイムに把握するためのハードルは、当初想定の2〜3倍になります。
+ 未解決の矛盾 3件（3/5のMTGで確認したい事項）
1. 「FAXが半分、ANDPADが半分」というお話は、受発注ではなくチャット連絡のことでしょうか？
2. 基幹が「クラウド」なのに「つなぎ口なし」ということは、サーバーを借りて動かしている形態でしょうか？
3. 自社見積システムは基幹システムの一部機能でしょうか、それとも完全に別のシステムでしょうか？
Structural Issues

個別の課題ではなく、構造として繋がっている

ニッカホームが直面している課題は、それぞれ独立しているように見えて、実は1つの構造から派生しています。

原価管理の悪循環
発注がFAX・電話（紙ベース）で行われている
請求書が届くまで原価が分からない（2ヶ月の遅延）
経営判断が常に2ヶ月遅れで行われている
利益率の見えない案件がそのまま進行し続ける
問題に気づくのは粗利が確定した後。手を打てるタイミングを逃している
ナレッジ消失の連鎖
ベテラン社員が離職する（年間15人、離職率25〜33%）
その人だけが持っていた経験・判断基準が消える
新人が「誰に聞けばいいか」を見つけられない（電話で30分待ち）
施工現場でのトラブルが増加する（手戻り工事＋お客様への補償）
1件あたり平均150万円の損失。ベテランがいた頃は起きなかった問題が常態化する
データ分散の罠
6つのシステムが完全にサイロ化している（データ連携ゼロ）
月次レポートが手作業（37営業所 x 月5時間）
管理職の時間が集計作業に消えている
本来の営業・マネジメント業務が後回しになっている
リフォーム検討者の問い合わせ経路は多層的：
紹介(21.8%) → ポータルサイト → Google検索 → GBPで口コミ・写真を最終確認 → 問い合わせ
GBPは「集客エンジン」ではなく信頼の最終確認ツール（衛生要因）
37店舗のGBPがバラバラに管理され、口コミ放置の店舗もある
他チャネルで興味を持った見込み客が、GBPの印象で離脱している可能性
これらは別々の問題に見えますが、根っこは同じ構造にあります。「データが繋がっていない」「紙が残っている」「経験知が個人に閉じている」。この3つが相互に悪循環を生んでいます。
If Nothing Changes

現状維持の場合に予想されること

ニッカホームが167店舗への拡大を目指す中で、現在の仕組みのまま進んだ場合、2〜3年後にどうなるかを整理しました。

原価管理

2ヶ月の遅延構造が167店舗に拡大します。把握できない案件数が現在の1.4倍に。年間で数千万円の利益が「見えないまま」流出し続ける計算になります。

ナレッジ

3年間で45人分の経験知が消失します。新人がトラブル対応に追われることが常態化し、手戻り工事のコストが年々積み上がります。

集客

リフォーム検討者はチラシや紹介で知った会社を、最終的にGBP（Googleマップ）の口コミで確認します。口コミ放置・低評価の店舗は、せっかくの見込み客を競合に流出させ続けます。

システム基盤

基幹システムは2015年に構築され、すでに10年を超えます。保守ベンダーの対応リスクとセキュリティリスクが年々高まります。

167店舗への拡大は、現在の仕組みのまま実現しようとすると、問題が店舗数に比例して拡大します。逆に言えば、今この仕組みを変えれば、拡大するたびに効果も比例して大きくなります。

Short-term Actions

追加投資なしで確認できる4つのこと

Google Workspace の既存契約内で完結し、1〜3ヶ月で効果が数字で見える施策です。
中期投資（ANDPAD受発注など）の判断材料にもなります。

QW-1

月次速報の自動化

Before Excel手作業で月末5時間。37営業所の管理職が集計に追われる
After Looker Studioで自動更新。スマホからでも数字を確認できる
対象工数：37営業所 x 管理職1.5人 x 月5時間 x 時給3,000円 = 年間約1,000万円相当
前提：管理職の時給は3,000円で試算。自動化率は70%程度を想定しています。
QW-2

NotebookLMでナレッジ集約

Before ベテランに電話 → 出ない → 30分待ち。離職のたびにナレッジが消える
After スマホで質問 → 10秒で出典付き回答。経験知が組織に残る
離職15人/年 x 採用育成600万 x 復元率30% + トラブル防止150万/件 = 年間数千万円規模の影響
前提：育成コスト600万は業界平均。復元率30%は初年度の保守的な見積りです。NotebookLMは「資料に書いていないことは"わからない"と答える」設計のため、建設現場での安全性が高い仕組みです。
QW-3

Google ビジネスプロフィール 基盤整備

問い合わせの構造（住宅リフォーム推進協議会 調査より）
リフォーム検討者の情報収集：Web全般 63.4% ／ 紹介 21.8% ／ ショールーム 23.2% ／ チラシ 15.2%
→ 多くの場合、紹介・チラシ・ポータルサイトで会社を知った後、GBP（Googleマップ）で口コミ・写真を最終確認して問い合わせを判断。
→ GBPは「問い合わせを増やす攻めのツール」ではなく、「見込み客を取りこぼさないための信頼基盤」。
Before 37店舗がバラバラに管理。口コミ放置・情報未更新の店舗もある → 他チャネルで興味を持った見込み客が離脱
After 本部で品質を一括管理。口コミ返信・写真更新の最低ラインを全店舗で担保 → 機会損失を防止
投資額：月0〜5.6万円（GBP API無料枠 or Gyro-n MEO） → 月1件の取りこぼし防止で投資回収が成立する低リスク施策
GBP単体での問い合わせ増加効果を示す信頼できる統計データは存在しません（代理店の成功事例は生存者バイアスを含みます）。効果は「攻めのROI」ではなく「守りの機会損失防止」として評価すべきです。
なお、問い合わせ最大化には、紹介の仕組み化・自社サイトSEO・ポータルサイト活用などGBP以外のチャネル投資が別途必要です。
QW-4

業務フロー可視化

Before 業務フローが暗黙的。部署ごとに認識がバラバラ
After 全社共通の「業務地図」ができる。ボトルネックと改善ポイントが明確に
中期投資（ANDPAD受発注など）の判断精度が向上します
定量化は困難ですが、業務フローを可視化せずに投資判断すると、投資の失敗確率が大幅に上がります。

設計思想：「判断ゲート」で進める

Phase 1 は追加投資ゼロ。Phase 2 で初めて有償ツール（ANDPAD受発注 等）の判断。Phase 3 は Phase 1-2 の実績をもとにした中長期投資判断。
いきなり大型投資を求めないため、リスクを最小化しながら段階的に拡大できます。
Discussion Points

3/5のMTGで議論したいポイント

想定される論点と、それぞれの「考え方」を整理しました。

営業本部長の視点：投資対効果の試算は妥当ですか？
本レポートの試算はすべて「前提」を明記しています（管理職時給3,000円、離職率25〜33%、成約率5〜10%など）。3/5のMTGでニッカホームの実態に合わせて数字を補正し、より精度の高い試算に更新することを考えています。
営業所長の視点：現場の負担が増えませんか？
新しいツールを導入するのではなく、すでに使っているGoogle Workspaceの活用方法を変えるだけです。研修は1時間で完了する設計にしています。「覚えることが増える」のではなく「今やっている手作業が減る」方向です。
システム担当の視点：保守の負担は増えますか？
短期4施策はすべてGoogle Workspace内で完結します。サーバーの管理や新しいシステムの保守は発生しません。既存のGoogle管理者権限で運用できます。
経営層の視点：中期・長期で費用はどれくらいかかりますか？
短期は追加費用ゼロです。中期以降は短期施策の結果データをもとに判断します。「後から見えないコストが膨らむ」設計にはしません。中期のANDPAD受発注は月額制で、段階的な導入が可能です。
協力会社への影響はありますか？
短期施策では協力会社への影響はありません。中期のANDPAD受発注を検討する段階で、協力会社側もアプリ（無料）を利用していただく必要が出てくる可能性があります。
Next Step

3/5以降の進め方として、3つの選択肢があります

いずれの場合も、次のアクションは3/5のMTGでお話しして決められればと考えています。

Option A

まず1〜2施策を試す

すぐ出せる成果の中から1〜2施策を選び、パイロット営業所で1ヶ月試してみる。効果を数字で確認してから、全社展開を判断する。

Option B

業務フロー可視化を先に

現状の業務フローを全社で共有するワークショップを先に実施する。全体像を把握してから、すぐ出せる成果に着手する。

Option C

もう少し深掘りする

システム担当や現場の所長への追加ヒアリングで診断を深め、より精度の高い戦略を策定してから判断する。
