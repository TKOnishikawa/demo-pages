System Architecture

システム要件（全体アーキテクチャ）

4層アーキテクチャと信頼度ベースのルーティング設計。各コンポーネントは独立して稼働し、段階的に統合する。

システム全体構成

Input

入力層 — FAX受信 → PDF取得

複合機がFAXを受信 → PDF化 → メール転送またはGoogle Drive保存。手動アップロードにも対応。対応形式: PDF / PNG / JPG / TIFF。

Processing

処理層 — Gemini 2.5 Pro Vision API

PDFをGemini APIに送信 → JSON構造化データを取得。抽出プロンプトで13項目のメタデータ + items配列を生成。Temperature 0.1で決定的出力。

Routing

判定層 — バリデーション + 信頼度ルーティング

ISBN-13チェックデジット検証、数量異常値検知（1-999範囲チェック）、書名存在チェック。信頼度スコアに基づく3段階ルーティング。

Output

出力層 — Google Sheets / kintone / 基幹システム

Phase 1: Google Sheets（自動書き込み + 色付きフラグ）。Phase 2: Streamlit UIでHuman-in-the-Loop確認。Phase 3: kintone → 基幹システム連携。

信頼度ルーティング（Confidence Routing）

≥90%
自動承認
Sheets直接書き込み。ISBNチェック済み。目標: 全件の60%以上
70-90%

<70%

Validator

order_validator.py — バリデーション + 信頼度判定

ISBN-13チェックデジット検証（mod-10アルゴリズム）、数量異常値検知、信頼度レベル判定を実行。

Validation Rules
ISBN-13:   ハイフン除去 → 13桁数字チェック → mod-10検証
数量:      null → NG | <=0 → NG | >999 → NG | >100 → 要確認
書名:      2文字未満 → NG
信頼度:    ≥0.90 → high | ≥0.70 → medium | <0.70 → low
           警告あり + high → medium に降格
