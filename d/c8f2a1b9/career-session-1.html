3年9ヶ月の映像制作PM。赤文字（夢中）の業務を分析すると ——

赤文字 = 自己機能が発火していた業務

赤文字の業務本質自己機能との接続

KEY FINDING

夢中になっていたのは「進行管理」ではなく「引き出す → 形にする → 橋渡しする」。
「現場の進行」「実行予算」「作業の分担」は赤でも青でもない = やれるけどワクワクしない。

INSIGHT

自己機能は、PMの「How管理」よりプロデューサー/営業の「Why・What」に圧倒的にフィット。
ソーダの赤文字がそれを証明。「引き出す」「具現化する」「橋渡し」はWhy・Whatの仕事。

「Howが苦手」は本当か？ — ADHD特性の観点から

ADHD特性的に「管理」は苦手な傾向。ただしりょーたさん個人で見ると：

  • ソーダで3年9ヶ月PMを続けている（本当に苦手なら持たない）
  • 「現場の進行」「実行予算」は青文字ではなく無色（やれる）
  • 「カット表作成」は赤文字 = 「管理」ではなく「構造化」として楽しんでいた

正確な表現: 「Howもやれるけど、Why・Whatの方が圧倒的にワクワクする」。
「Howしかない仕事」は避けるべきだが、「Why・Whatがメインで、Howも一部含む仕事」は問題ない。

SECTION 04

4年のカフェ経験 — 最大の突っ込みポイントをどう乗り越えるか

正直に言えば、ここが選考で一番突っ込まれるところ。だからこそ「戦う場所」を選ぶ戦略が必要。

面接官の本音（事実として受け止める）

「映像PMの後に4年もカフェとセラピスト。デザイン業界と接点がない。なぜ今さらこの業界に？」
これは必ず聞かれる。しかも最も答えにくい質問。ここを正面突破するのではなく、「この経験が武器になる場所で戦う」のが戦略。

事実ベースで見直すと

経験実は何を鍛えていたか活きる場所
カフェファソン（スペシャルティ）味覚・風味の観察力。好みを見抜いてマッチングコーヒー業界なら即戦力
CITAN / BPJ（ホテル内カフェ）宿泊×コミュニティの体験設計ホスピタリティ系で原体験
Garden（オフィスカフェ）法人への飲食サービス導入支援法人向けサービスで実務経験
エーワン（セラピスト）身体の観察→最適アプローチの選択ウェルネス系で専門知識
STRATEGY

4年のカフェ経験を「弱み」のまま戦うのは愚策。
デザイン会社にPMで応募すると、この4年は「ブランク」でしかない。
しかしカフェ・飲食・ホスピタリティ系の事業会社に応募すれば、同じ4年が「現場のリアルを知っている」という最大の武器に変わる。
弱みをなくすのではなく、弱みが武器に変わる場所で戦う。これが戦略。

4年ブランクの説明ストーリー — 3パターン

A: 正直路線

「正直に言うと、定まっていませんでした。ただ、どの仕事でも一番力を発揮していたのは"相手を観察して最適なものを提案する瞬間"でした。この4年は遠回りでしたが、自分の核を見極められた期間です。」

B: 未来志向＋行動証拠

「迷走した期間は認めます。ただ今は方向性が明確です。御社の業界構造から業務プロセスまで徹底リサーチした上で来ています。」→ リサーチ資料を見せる

C: カフェ経験を武器に（TYPICA・トランジット等向け）

「スペシャルティコーヒーの世界に惹かれ、複数のカフェで働きました。この現場経験があるからこそ、御社のブランドを"中の人"の視点で理解し、企画に落とし込めます。」→ カフェ系企業向けではこれが最強

IMPORTANT

生成AI（Midjourney、Canva AI等）の進化で、デザインの「叩き」は誰でも作れる時代。

企画段階で「叩き」を自分で作れるプロデューサーは、デザイナーとの対話の質が全く違う。
これは「デザイナーを置き換える」のではなく、「デザイナーとのコミュニケーションを爆速化する」スキル。

29歳。ポテンシャル採用のラスト。走りながら整える。

3つの必須ポイント

#ポイントなぜ大事か
1ソーダの3年9ヶ月を「武器」として語る赤文字の業務中心に。「進行管理者」ではなく「引き出す→形にする人」として
24年間のストーリーを準備する応募先によって最強のストーリーが変わる。パターンを持っておく
3転職活動しながら実績を1つ作る知人のカフェに「ブランド診断」を持ちかけるだけでポートフォリオが1件できる

カジュアル面談 = 自分の仕事のデモンストレーション

面談に仮説企画書を持っていく。「もし自分が御社で働くなら」を1枚にまとめる。

  • ずれていてもいい。「この人は入社前からうちの課題を考えてる」という本気度の証明
  • これ自体が「観察 → 引き出す → 設計 → 届ける」のデモンストレーション
  • 他の応募者は絶対やってない。圧倒的に差がつく
仮説企画書の作り方

構成（1枚）

  1. 企業の現状理解（AIリサーチから）
  2. 自分が感じた「もったいない」ポイント
  3. 施策案（あくまで叩き）
  4. 質問（「この方向性って実態と合ってますか？」）

例: TYPICA向け

「ロースターが豆を選ぶときの"体験"をもっとリッチにできるのでは。生産者のドキュメンタリー短編映像をプラットフォームに掲載する。ロースター同士のコーヒー交換会イベントを月1で開催する。」
→ 映像PM経験×バリスタ知識の両方が自然に入る

今週

気になる会社にカジュアル面談を申込む。知人のカフェに「ブランド診断」声かけ

来週

面談先の企業をAIリサーチ → 仮説企画書1枚。職務経歴書を赤文字中心に書き直す

2-3週後

カジュアル面談実施。知人店舗のヒアリング → 提案書作成

1ヶ月後

正式応募開始。ポートフォリオに提案書を組み込み
